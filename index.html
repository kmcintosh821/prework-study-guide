HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements that contain other elements are "parent" elements, whereas the elements inside them are "child" elements.

Available head elements:

  • meta element: lists metadata, such as charset, http-equiv, or viewport
  • title element: the name of the page, as displayed in a tab
  • style element: defines CSS styles
  • base element: the base URL for a webpage
  • link element: connects an external resource to the HTML doc
  • script element: embeds client-side scripts in the HTML document

Available body elements:

  • h1 thru h6 elements: various levels of heading text, from largest to smallest (on this page, the page title is a h1 header, "HTML" is a h2 header, and "Available body elements" is a h3 header!)
  • p element: paragraph text
  • ul and ol elements: unordered (bulleted) and ordered (numbered) lists (these notes are in ul tags!)
  • li element: a list item, used as a child element of the above list elements (like this one!)
  • img element: an image (the src attribute points to the URL for the link)
  • br element: a line break, or a line of empty space (that's why these notes aren't all scrunched on each other!)
  • a element: an anchor element, which creates links to certain webpages (points to an URL using the href attribute)

CSS

  • CSS rules change the appearance of HTML elements.
  • They can be applied to elements across the whole page, by using the element without any brackets or punctuation as your selector.
  • They can also affect personally-defined classes, by using the class name with a period (.classname) as the selector.
  • In order: selector {declaration-property-1: declaration-value-1; property-2: value-2;}

Potential CSS rule properties:

  • Margin, border, padding: the structural parts of a CSS "box". Can be defined by pixels or percentages!
  • Width, height: the size of the actual content, the centermost element of the CSS box. Also can be pixels or percents!
  • display: How the content fits into the box -- when an image is set to display: block, it starts on a new line
  • -left, -right, -top, -bottom: Added to structural parts to define more specific aspects of them, such as margin-right or padding-left
  • font-size: The size of the font, by height in pixels.
  • color, background-color: Defines the text or background's color! Can be semantic colors or hex codes.
  • box-shadow: Makes the box pop! Can be defined by x and y offsets, as well as the color of the shadow. Blur and spread radius are also potential attributes!

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • -b: specifies the branch
  • git add -A: stages all files in the working branch, in order to be committed
  • -A: denotes "all"
  • git commit -m "message": commits anything staged, with the denoted message
